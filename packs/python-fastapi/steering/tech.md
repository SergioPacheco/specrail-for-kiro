# Tech Steering

## Language and runtime
- Python 3.11+ (prefer 3.12+)
- FastAPI 0.100+ (Pydantic v2 compatible)
- ASGI server: uvicorn with `--workers` for production, or gunicorn + uvicorn workers

## Core dependencies
- Web: FastAPI
- Validation: Pydantic v2 (built into FastAPI)
- ORM: SQLAlchemy 2.x with async support (`asyncpg` driver)
- Migrations: Alembic
- HTTP client: `httpx` (async-native)
- Task queue: Celery + Redis or `arq` for async tasks

## Database
- PostgreSQL 15+ via `asyncpg` (async) or `psycopg` (sync)
- Connection pool: SQLAlchemy async engine with pool settings
- Migrations: Alembic with `--autogenerate` for schema diffs

## Build and packaging
- Dependency management: `uv` (preferred) or `poetry`
- Lock file committed to git (`uv.lock` or `poetry.lock`)
- Docker: multi-stage build, `python:3.12-slim` base image
- Type checking: `mypy --strict` or `pyright`

## Key libraries
- Logging: `structlog` for structured JSON logging
- Testing: `pytest` + `pytest-asyncio` + `httpx` + Testcontainers
- Mapping: manual or `pydantic` model validators
- API docs: auto-generated by FastAPI (Swagger UI at `/docs`)
- Settings: `pydantic-settings` for typed environment config

## Deployment
- Container-based: Docker → Kubernetes or ECS
- Health: `/health` endpoint returning `{"status": "ok"}`
- Graceful shutdown: uvicorn handles SIGTERM
- Metrics: `prometheus-fastapi-instrumentator` or custom middleware

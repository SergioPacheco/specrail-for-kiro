## Module boundaries
- Routers: receive HTTP, validate via Pydantic, delegate to service, return schema. No business logic.
- Services: business logic, transaction coordination. Injected via `Depends()`.
- Repositories: SQLAlchemy queries only. Return ORM models.
- Schemas: Pydantic models for API input/output. Separate from ORM models.
- Models: SQLAlchemy ORM models. Never returned directly in API responses.

## Naming conventions
- Files: `snake_case.py` вЂ” `user_service.py`, `order_repo.py`
- Classes: `PascalCase` вЂ” `UserService`, `CreateUserRequest`, `UserResponse`
- Functions: `snake_case` вЂ” `get_user_by_email`, `create_order`
- Constants: `UPPER_SNAKE_CASE` вЂ” `MAX_PAGE_SIZE`, `DEFAULT_TIMEOUT`
- Routes: `kebab-case` вЂ” `/api/v1/user-accounts`
- Migrations: auto-generated by Alembic with descriptive message
